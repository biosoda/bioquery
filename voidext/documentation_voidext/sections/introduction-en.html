Introduction back to ToC

This documentation describes a RDF schema called Extended Vocabulary of Interlinked Datasets (VoIDext) to enhance data interchange between distributed data sets. Although VoID RDF schema provides the void:linkPredicate term to define by which property two datasets are linked, solely this term is not enough to consider all possible virtual links (see definition in Subsection 1.1) between two datasets. Because of this, we extended and adapted the VoID vocabulary to cover more possibilities of liking two distributed datasets of RDF triples. In Section 3, we depict how to use this RDF schema in order to define the virtual links. VoIDext is fully relied on the RDF schema, thus no OWL terms are used. In addition, VoIDext does not contain all VoID terms because VoIDext focus on the virtual links description. This does not impedes to re-use other VoID RDF schema terms along with VoIDext.

What is a virtual link?

A virtual link is an intersection data point between two or more data stores. The links are required in order to enable performing federated queries, given that they act as join points between the federated sources.
