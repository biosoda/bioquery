VoIDext RDF schema: Description back to ToC

Once the data stores are accessible through SPARQL endpoints, we can exploit means to link them at the data level. To do so, we can identify common class instances and literals (e.g. strings) in order to establish virtual links. Figure 1 illustrates virtual links among UniProt, Bgee and OMA datasets. For example, OMA and Bgee describe complementary information about common genes (instances of the orth:Gene class), as well as taxa (instances of the up:Taxon class), both of which can serve as virtual links to connect the two data sources. Below an example by using the Turtle RDF syntax of how to describe shared instances of the same class in different datasets with VoIDext terms. In this example, OMA and Bgee datasets share the same instance Internationalized Resource Identifiers (IRIs). Indeed, OMA instances are reused as part of the Bgee RDF graph rather than creating new Bgee ones. 

  bioquery:OMA_BGEE_2 a voidext:SharedInstanceSet; #rdfs:subClassOf voidext:VirtualLinks .
            void:target bioquery:OMA;
            void:target bioquery:BGEE;
            voidext:sharedInstanceType orth:Gene.
  
  bioquery:OMA_BGEE_3 a voidext:SharedInstanceSet;
             void:target bioquery:OMA;
             void:target bioquery:BGEE;
             voidext:sharedInstanceType up:Taxon.

Thus, orth:Gene instances are intersection nodes (i.e. virtual links) between the Bgee and OMA graphs. Figure 1 illustrates a shared orth:Gene instance between OMA and Bgee graphs as a grayscale filled circle.

In addition to virtual links between instances of the same class, we can generalise virtual links into two different types: (i) IRI based (e.g. an instance), and (ii) data literal based (e.g. a label, such as "Homo sapiens" or "HBB"). For example, cross-reference IRIs serve as virtual links of type (i) among the data stores. Concretely, let us consider cross-references to Ensembl genes. On the one hand, the Ensembl gene IRI is assigned to up:transcribedFrom OWL object property in UniProt RDF store. This IRI is illustrated as a filled black circle in Figure 1. On the other hand, we reuse the same IRI as value of the lscr:xrefEnsemblGene OWL annotation property in the Bgee and OMA RDF graphs. This example is better illustrated in Figure 2 where this virtual link of type (i) is defined as an instance of voidext:ComplexLinkSet that is a subclass of voidext:VirtualLinks. To illustrate type (ii), we can mention the rdfs:label and skos:prefLabel properties that assert the same gene names (e.g. "HBB") to an instance of orth:Gene in the Bgee graph and up:Gene in the UniProt RDF graph, respectively. In Figure 1, these literals are represented as rectangles.

The example below defines with the Turtle RDF syntax some of the instances shown in Figure 2. This example illustrates how to instantiate virtual links of type voidext:ComplexLinkSet with the VoIDext RDF schema. 
bioquery:UNIPROT_OMA_2 a voidext:ComplexLinkSet; ### The filled gray circle in Figure 2
			void:target bioquery:OMA;
			void:target bioquery:UNIPROT;
			voidext:intersectAt bioquery:UNIPROT_EBI-ENSEMBL ;
			voidext:intersectAt bioquery:OMA_EBI-ENSEMBL ;
			voidext:intersectionType voidext:OBJECT_OBJECT .

bioquery:UNIPROT_EBI-ENSEMBL a  void:Linkset; ### The filled black circle in Figure 2
             void:subjectsTarget bioquery:UNIPROT;
             void:objectsTarget bioquery:EBI;
             void:linkPredicate up:transcribedFrom;
             voidext:linkPredicateDomain up:Transcript_Resource;
             voidext:linkPredicateRange obo:SO_0000704.
			
bioquery:OMA_EBI-ENSEMBL a void:Linkset; ### The white circle in Figure 2
             void:subjectsTarget bioquery:OMA;
             void:objectsTarget bioquery:EBI;
             void:linkPredicate lscr:xrefEnsemblGene;
             voidext:linkPredicateDomain orth:Gene;
             voidext:linkPredicateRange obo:SO_0000704.	

bioquery:OMA a  void:Dataset.
bioquery:UNIPROT a  void:Dataset.
bioquery:EBI a  void:Dataset.
For further instantiation examples serialised with Turtle syntax, please refer to the BioQuery RDF federation metadata file.
